## 5) Gameplay recording

After training the RL agents and starting your VLM server:

```bash
python record_gameplay.py
```

This records one gameplay video for each agent and writes:
- `dqn_gameplay.mp4`
- `ppo_gameplay.mp4`
- `vlm_gameplay.mp4`

### Optional custom command

```bash
python record_gameplay.py \
  --dqn-path best_dqn.zip \
  --ppo-path best_ppo.zip \
  --api-base http://localhost:11434/v1 \
  --model Qwen/Qwen3.5-2B \
  --fps 30
```

### What is more robust in this pass
- The recorder now checks that required model files exist before starting.
- It streams frames directly to disk instead of keeping everything in memory.
- It exposes CLI flags for model paths, output names, FPS, max steps, and seed.
- It uses a dedicated raw RGB render env for the saved video while the RL model still receives its usual preprocessed stacked observations.
- It reports a clearer failure if a video is empty or rendering fails.

## 6) Reflection answers

These are polished draft answers. Update them to match your actual final observations.

### (a) What was the single most impactful change you made to improve your DQN or PPO agent, and why do you think it helped?
The single most impactful change was increasing the training budget to 10 million timesteps while selecting the best checkpoint using a separate evaluation environment. In Atari, performance often improves meaningfully late in training, so a short run can underfit the game and make good hyperparameters look worse than they really are.

### (b) Did your VLM agent perform above a random baseline? What specific behavior (or lack thereof) did you observe?
The VLM sometimes appeared better than random because it could detect the player ship, obvious projectile threats, and situations where firing was immediately useful. Its main weakness was control consistency: it still reacted slowly, occasionally oscillated, and struggled to maintain strong long-horizon positioning without guardrails.

### (c) The VLM received higher-resolution color images while the RL agents saw 84×84 grayscale. Why wasn’t more information enough?
More pixels did not solve the core problem, which is sequential control under tight timing constraints. The RL agents learned a reward-optimized policy from millions of interactions, while the VLM had to infer a control policy zero-shot from images and prompt text, so the bottleneck was decision learning rather than raw visual detail.
